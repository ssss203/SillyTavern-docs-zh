# 🎛️ 通用设置（采样参数）
> 💡 **肆说**：这些参数控制AI"怎么写"。新手只需要关注温度和回复长度，其他保持默认就好。

这些设置控制用大语言模型生成文本时的采样过程。含义在所有支持的后端中通用。

## 上下文设置

### 回复长度（Response Tokens）
> 💡 **肆说**：AI最多写多少token。想写长就调高，但太长AI可能凑字数。一般300-800就够了。
API 生成的最大 token 数。越长越慢。支持流式输出（Streaming）时回复逐段显示，关闭时等全部生成后一次显示。

### 上下文长度（Context Tokens）
> 💡 **肆说**：AI的"记忆容量"。越大AI能记住越多的聊天历史，但也越贵/越慢。
SillyTavern 发送给 API 的提示词最大 token 数，减去回复长度。上下文包括角色信息、系统提示词、聊天历史等。虚线以上消息不发送给 AI。

## 采样器参数

### 温度（Temperature）
> 💡 **肆说**：最关键的参数！0.7-1.0是安全区。想AI更"正经"调低，更"放飞"调高。别超过1.5不然会胡言乱语。
控制 token 选择的随机性。低温(<1.0)更可预测，高温(>1.0)更有创意。设1用原始概率。

### 重复惩罚（Repetition Penalty）
> 💡 **肆说**：AI开始重复说话了？调高这个，一般1.05-1.15就够。太高AI说话会很怪。
根据 token 在上下文中的出现频率惩罚来抑制重复。设1禁用。
- **范围（Range）**：纳入计算的 token 数。设0禁用。
- **斜率（Slope）**：惩罚在末尾效果更强。设0禁用。

### Top K
可选顶部 token 的最大数量。设0禁用。

### Top P（核采样，Nucleus Sampling）
> 💡 **肆说**：跟温度类似但控制方式不同。0.9-0.95是常用值，一般不用动。
累加顶部 token 概率达到目标百分比。设1禁用。

### Typical P
按偏离平均熵的程度优先排序 token。设1禁用。

### Min P
> 💡 **肆说**：推荐跟温度配合使用！0.05-0.1是好起点，能过滤掉AI的"胡说八道"选项。
相对顶部 token 切断低概率 token。低值(0.1-0.01)效果最好。设0禁用。

### Top A
基于最高 token 概率平方设阈值。设0禁用。

### 尾部自由采样（TFS, Tail Free Sampling）
分析概率变化率的导数搜索低概率尾部。越接近0丢弃越多。设1禁用。

### 平滑因子（Smoothing Factor）
二次变换增高低概率、降低低概率。不用截断采样器时效果最好。设0禁用。

### 动态温度（Dynamic Temperature）
根据顶部 token 可能性动态缩放温度。取消勾选禁用。

### Epsilon 截断（Epsilon Cutoff）
概率下限，低于此的 token 被排除。单位1e-4，合理值3。设0禁用。

### Eta 截断（Eta Cutoff）
Eta 采样技术的主要参数。单位1e-4，合理值3。设0禁用。

### DRY 重复惩罚（DRY Repetition Penalty）
> 💡 **肆说**：比普通重复惩罚更智能，专门防止AI"复读机"式重复。乘数设0禁用，推荐开启。
惩罚会把输入末尾扩展成已出现序列的 token。可加序列中断列表。设乘数0禁用。

### 排除顶部选择（XTC）
移除最可能 token 而非修剪最不可能的。设概率0禁用。

### Mirostat
让输出困惑度匹配输入困惑度。模式：0=禁用, 1=Mirostat 1.0(仅llama.cpp), 2=Mirostat 2.0。

### 束搜索（Beam Search）
贪心暴力算法，同时扩展多个候选序列找最可能序列。

### Top nσ
基于统计特性过滤 logits。保留最大 logit 值 n 个标准差内的 token。
