# 指令模式（Instruct Mode）
> 💡 **肆说**：指令模式是给 Text Completion API 用的。如果你用 Chat Completion（OpenAI/Claude等），这个跟你没关系，跳过。

指令模式允许你调整针对各种提示词格式训练的指令遵循模型（如Alpaca、ChatML、Llama2等）的提示词构建方式。

##### 适用于：文本补全API

对于聊天补全API中的等效设置，请使用[提示词管理器](../prompts/prompt-manager.md)。

## API支持

### 文本补全API

完全支持。包括文本补全下的所有来源、KoboldAI Classic和AI Horde。

### 聊天补全API（OpenAI、Claude等）

**不支持（也不需要）**。它们使用完全不同的提示词构建器。

### NovelAI

虽然技术上支持，但NovelAI的模型都没有被训练来理解指令格式，使用指令模式将导致输出**质量下降**。

## 指令模式设置

### 系统提示词

系统提示词现在是一个独立实体。详见[高级格式化](advancedformatting.md#系统提示词)页面。

### 模板

提供带有一些知名指令模型序列的现成模板。*更改模板会重置未保存的设置！*

### 激活正则表达式

如果定义为有效的正则表达式，当连接到模型且其名称匹配时，将自动选择此模板。

### 用换行符包裹序列

每个序列文本在插入提示词时将用换行符包裹。Alpaca及其衍生版本需要此设置。

### 在序列中替换宏

如果启用，消息包裹序列中定义的已知宏替换将被替换。特殊的{{name}}宏可以在消息前缀中使用。

### 包含名称

- **从不**：不添加名称前缀
- **群组和历史人设**：仅为群组角色和历史人设添加
- **总是**：总是添加

### 序列：故事字符串包裹

定义当位置设置为"默认"时，故事字符串如何被包裹。包括故事字符串前缀和后缀。

### 序列：聊天消息包裹

定义属于不同角色的消息如何被包裹。所有前缀序列也将自动用作停止字符串。

- **用户消息前缀/后缀**：插入在用户消息前后
- **助手消息前缀/后缀**：插入在助手消息前后
- **系统消息前缀/后缀**：插入在系统消息前后
- **系统与用户相同**：如果勾选，系统消息使用用户角色序列

### 其他序列

- **第一个助手前缀**：插入在聊天历史第一个助手消息之前
- **最后一个助手前缀**：插入在最后一个助手消息之前
- **系统指令前缀**：后台生成时使用
- **用户填充消息**：聊天历史不以用户消息开始时插入
- **停止序列**：表示回复结束的文本
